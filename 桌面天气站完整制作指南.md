### Raspberry Pi Pico 2 W

本项目使用 **Pico 2 W**，基于 RP2350 芯片，性能和内存大幅提升。

| 特性 | Pico W (RP2040) | **Pico 2 W (RP2350)** |
|------|-----------------|----------------------|
|------|-----------------|----------------------|
| CPU | 双核 M0+ 133MHz | **双核 M33 150MHz** |
| RAM | 264KB | **520KB** |
| Flash | 2MB | **4MB** |
| WiFi | CYW43439 | CYW43439（相同） |
| 蓝牙 | BT 4.2 | **BT 5.2** |
| GPIO | 26 个 | 26 个（**引脚完全兼容**） |
| 价格 | ¥35 | **¥45** |

**为什么选 Pico 2 W：**
- 520KB RAM 足够驱动 3.5 寸 480×320 彩色屏幕
- Cortex-M33 比 M0+ 更快，动画更流畅
- 引脚与旧 Pico W 完全兼容，接线不变
- Rust Embassy 框架官方支持成熟

```powershell

winget install Rustlang.Rustup

# 方式二：去官网下载 rustup-init.exe
```

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

安装完成后确认：

```bash
rustc --version

```

### 3.2 安装嵌入式工具链

```bash
# RP2350 是 Cortex-M33，编译目标为 thumbv8m.main-none-eabihf
rustup target add thumbv8m.main-none-eabihf

# 烧录工具（推荐 probe-rs，也可用其他方式）
cargo install probe-rs-tools

# 栈溢出保护（Pico 双核推荐）
cargo install flip-link
```

### 3.3 Windows 额外步骤：安装 USB 驱动

probe-rs 需要 WinUSB 驱动：

1. 下载 [Zadig](https://zadig.akeo.ie/)
2. 按住 Pico 的 BOOTSEL 按钮，插入 USB
3. 在 Zadig 中选择 Pico 设备
4. 选择 **WinUSB** 驱动
5. 点击 Install Driver

### 3.4 验证环境

```bash
rustc --version
cargo --version
probe-rs --version
```

三个命令都有正常输出，环境就搭好了。

### 4.2 BME280 温湿度气压传感器

#### BME280 是什么？

BME280 是 Bosch 出品的数字环境传感器，集成温度、湿度、气压三合一测量。I2C 接口，和 DS3231 共用同一条总线。

**规格参数：**

| 参数 | 值 | 说明 |
|------|-----|------|
| 供电电压 | 1.71V ~ 3.6V | 本项目用 3.3V |
| 温度范围 | -40°C ~ +85°C | 精度 ±0.5°C（25°C 时 ±1°C） |
| 湿度范围 | 0% ~ 100% RH | 精度 ±3% RH |
| 气压范围 | 300 ~ 1100 hPa | 精度 ±1 hPa |
| I2C 地址 | 0x76 或 0x77 | 取决于 SDO 引脚接法 |
| 响应时间 | 几毫秒 | 比 DHT11 快 100 倍 |

**买哪种？**

推荐买 **带 PCB 的六脚模块**（GY-BME280），引脚标注清晰：

```
推荐：GY-BME280 模块（¥18）
┌─────────────────┐
│  ┌───────────┐  │  ← Bosch BME280 芯片
│  │  BME280   │  │     已焊好排针
│  └───────────┘  │     已内置电平转换
│  VCC GND SDA SCL│     3.3V/5V 兼容
│  SDO CSB       │
└─────────────────┘
  六脚模块
```

#### 接线

| BME280 引脚 | Pico 引脚 | 说明 |
|-------------|----------|------|
| VCC | 3V3 (Pin36) | 供电 |
| GND | GND (Pin38) | 接地 |
| SDA | GP0 (Pin1) | I2C 数据线（与 DS3231 共用） |
| SCL | GP1 (Pin2) | I2C 时钟线（与 DS3231 共用） |
| SDO | GND 或 3V3 | 地址选择：接 GND = 0x76，接 3V3 = 0x77 |
| CSB | 3V3 | I2C 模式下接 3V3（SPI 模式才用） |

> **SDO 和 CSB 大多数模块已经内部处理好了，你只需要接 VCC/GND/SDA/SCL 四根线。**

#### I2C 地址

BME280 默认地址为 `0x76`（SDO 接 GND）。代码中先尝试 `0x76`，如果读不到芯片 ID `0x60`，再试 `0x77`。

#### BME280 vs DHT11 为什么值得多花 ¥13？

| | DHT11 (¥5) | **BME280 (¥18)** |
|---|------------|-----------------|
| 温度精度 | ±2°C | **±0.5°C** |
| 湿度精度 | ±5% RH | **±3% RH** |
| 气压 | ❌ | **✅ 1 hPa 精度** |
| 接口 | 单总线（需上拉电阻） | **I2C（总线共用，稳定）** |
| 可靠性 | 偶尔读取失败 | **I2C 通信，几乎不会失败** |
| 芯片品牌 | 国产仿制品 | **Bosch（德国博世）** |
| 送礼体面度 | 一般 | **靠谱，不翻车** |

### 4.3 3.5 寸 ILI9488 TFT 彩色显示屏 (SPI)

| ILI9488 引脚 | Pico 2 W 引脚 | 说明 |
|-------------|---------------|------|
| VCC | 3V3 或 5V (Pin36/40) | 供电（模块通常支持 3.3V/5V） |
| GND | GND (Pin38) | 接地 |
| SCK | GP10 (Pin14) | SPI1 时钟 |
| MOSI | GP11 (Pin15) | SPI1 数据（主出从入） |
| CS | GP9 (Pin12) | 片选（低电平有效） |
| DC | GP8 (Pin11) | 数据/命令选择 |
| RST | 3V3 (Pin36) | 复位（接高电平，模块内部有复位电路） |
| LED | 3V3 (Pin36) | 背光（接高电平常亮） |
| MISO | 不接 | 本项目不需要读取屏幕数据 |

> **注意：** ILI9488 模块引脚顺序可能和上面不完全一样，以模块上标注的丝印为准。有些模块还多一个 `T_CS` 引脚（触摸片选），不用的话悬空或接高电平。

**ILI9488 模块特点：**
- 分辨率 480×320，65K 色彩色 TFT
- 自带显存（RAM），Pico 不需要帧缓冲
- SPI 接口，接线简单（6 根线）
- 带触摸功能（电阻式/电容式，本项目不使用）

### 4.4 DS3231 实时时钟模块

| DS3231 引脚 | Pico 引脚 | 说明 |
|-------------|----------|------|
| VCC | 3V3 (Pin36) | 供电 |
| GND | GND (Pin38) | 接地 |
| SDA | GP0 (Pin1) | 与 BME280 共用 I2C |
| SCL | GP1 (Pin2) | 与 BME280 共用 I2C |

I2C 地址为 `0x68`。

> **I2C 总线说明：** BME280 和 DS3231 共享 GP0/GP1，它们有不同的 I2C 地址（`0x76` 和 `0x68`），可以同时工作，互不干扰。

### 4.5 蜂鸣器模块接线

| 蜂鸣器引脚 | Pico 2 W 引脚 | 说明 |
|-----------|--------------|------|
| VCC | **5V (Pin40)** | 5V 供电声音更大 |
| GND | GND (Pin38) | 接地 |
| I/O (信号) | GP6 (Pin9) | 高电平响，低电平停 |

> **有源蜂鸣器 vs 无源蜂鸣器：**
> - 有源蜂鸣器：给高电平就响，频率固定，只能做简单提示音。
> - **无源蜂鸣器**：需要方波驱动，可以发不同音调，能播放生日歌旋律。**本项目推荐用这个。**
>
> 代码中 `buzzer_melody()` 通过 GPIO 高低电平交替产生不同频率的方波来驱动无源蜂鸣器播放旋律。

### 4.6 TTP223 触摸传感器接线

| 触摸模块引脚 | Pico 引脚 | 说明 |
|------------|----------|------|
| VCC | 3V3 (Pin36) | 供电 |
| GND | GND (Pin38) | 接地 |
| I/O (信号) | GP7 (Pin10) | 触摸时输出高电平，松开变低 |

TTP223 是电容式触摸传感器，手指轻触金属面即可触发，不需要按压。手感比机械按键好很多，送女朋友很合适。

### 4.7 接线检查清单

- [ ] BME280、DS3231 的 SDA 接 GP0，SCL 接 GP1
- [ ] ILI9488 的 SCK 接 GP10，MOSI 接 GP11，CS 接 GP9，DC 接 GP8
- [ ] ILI9488 的 RST 和 LED 接 3V3（常亮常复位）
- [ ] 蜂鸣器 I/O 接 GP6
- [ ] 触摸传感器 I/O 接 GP7
- [ ] 所有模块 GND 共地
- [ ] BME280 和 DS3231 接 3V3 供电
- [ ] ILI9488 接 3V3 或 5V 供电（看模块标注）
- [ ] DS3231 模块上的电池已装好（CR2032）
- [ ] USB 线是数据线（不是纯充电线）

BME280 读取代码（内联驱动，异步 I2C）：

```rust
// 单次测量：触发测量 → 等待 50ms → 读取 8 字节原始数据 → 校准补偿
pub async fn measure(&mut self, bus: &I2cBus) -> Result<Measurements, i2c::Error> {
    self.write_reg(bus, 0xF4, 0x25).await?;       // 触发单次测量
    Timer::after(Duration::from_millis(50)).await;  // 等待测量完成
    let mut data = [0u8; 8];
    self.read_regs(bus, 0xF7, &mut data).await?;   // 读取原始数据
    let cal = self.cal.as_mut().unwrap();
    let t = Self::comp_t(raw_t, cal);   // 温度补偿
    let p = Self::comp_p(raw_p, cal);   // 气压补偿
    let h = Self::comp_h(raw_h, cal);   // 湿度补偿
    Ok(Measurements { temperature: t, pressure: p, humidity: h })
}
```

> **气压趋势智能表情：** 代码会持续记录气压历史（环形缓冲区，每 2 秒采样一次，保存最近 10 次），通过比较前半段和后半段的平均值来判断趋势：
> - 气压**上升**（差值 > 0.5 hPa）且气压 > 1015 hPa → 显示**晴天表情** ☀
> - 气压**下降**（差值 < -0.5 hPa）且气压 < 1000 hPa → 显示**悲伤表情** 😢（可能下雨）
> - 其他情况 → 使用默认的温度/时间/天气表情
>
> 这样即使没有联网，天气站也能根据气压变化"感知"天气变化趋势，让表情更智能。

```rust
    if state.wifi_connected && loop_start - state.last_ntp_sync >= 3600 {
        if sync_ntp(stack, &rtc, bus).await { state.last_ntp_sync = loop_start; }
    }

    // 7. 渲染主界面（圆形时钟 + 天气面板 + 表情 + WiFi 图标 + 日期）
```

## 7. 表情动画系统

### 7.1 表情设计原理

每个表情是一个 **8x8 像素的位图**，用 8 个 `u8` 字节表示，每字节代表一行，从高位到低位对应左到右。

```
示例：开心表情
行0: 0b00111100  →  ....██..
行1: 0b01000010  →  .█....█.
行2: 0b10010001  →  █..█...█
行3: 0b10000001  →  █......█
行4: 0b10100101  →  █.█..█.█  ← 嘴巴
行5: 0b10011001  →  █..██..█  ← 嘴巴
行6: 0b01000010  →  .█....█.
行7: 0b00111100  →  ....██..
```

动画通过 **两帧交替** 实现（如眨眼：睁眼/闭眼）。

### 7.2 全部表情一览

| 表情类型 | 枚举值 | 触发条件 | 描述 |
|---------|--------|---------|------|
| Happy | `FaceType::Happy` | 18-25°C / 上午/下午 | 微笑，偶尔眨眼 |
| VeryHappy | `FaceType::VeryHappy` | 25-30°C / 中午 | 大笑 |
| Sad | `FaceType::Sad` | 10-18°C / 阴天 | 撅嘴 |
| Cold | `FaceType::Cold` | < 10°C | 发抖（眼睛闪烁） |
| Hot | `FaceType::Hot` | 30-35°C | 出汗（嘴巴变化） |
| Angry | `FaceType::Angry` | > 35°C | 生气（皱眉） |
| Sleepy | `FaceType::Sleepy` | 早晚/深夜 | 打哈欠 |
| Surprised | `FaceType::Surprised` | 预留（未使用） | 惊讶（眼睛变大） |
| Sunny | `FaceType::Sunny` | 晴天 | 眯眼微笑 |
| Rainy | `FaceType::Rainy` | 下雨 | 悲伤 + 雨滴 |

### 7.3 表情选择逻辑

优先级：**气压趋势 > 天气 > 温度 > 时间**

```rust
fn select_face(temp: f32, code: &str, hour: u8, trend: PressureTrend, press: f32, filled: bool) -> FaceType {
    // 1. 气压趋势直接决定表情（最高优先级）
    match code {
        "trend_rain" => return FaceType::Sad,    // 气压骤降 → 悲伤
        "trend_sun"  => return FaceType::Sunny,  // 气压飙升 → 晴天
        _ => {}
    }
    // 2. 气压历史填满后，根据趋势判断
    if filled {
        match trend {
            PressureTrend::Falling if press < 1000.0 => return FaceType::Sad,
            PressureTrend::Rising if press > 1013.0 => return FaceType::Sunny,
            _ => {}
        }
    }
    // 3. 天气代码匹配（下雨/下雪 → 悲伤脸）
    let wf = face_by_weather(code);
    match code {
        "d04"|"d05"|"d06"|..|"d17"|"d19" => wf,
        // 4. 极端温度优先，否则按时间选
        _ => if temp < 10.0 || temp > 35.0 { face_by_temp(temp) } else { face_by_time(hour) },
    }
}
```

## 8. 网络天气功能（Pico 2 W）

> 本项目使用 Pico 2 W，WiFi 芯片 CYW43439 集成在板上，通过 Embassy 框架直接调用。

### 8.1 WiFi 连接方式

Pico 2 W 的 CYW43439 芯片通过 SPI（PIO 模拟）与 RP2350 通信，Embassy 提供了完整的异步 WiFi 驱动：

```rust
// 初始化 CYW43 WiFi 芯片（PIO 模拟 SPI）
let (net_device, mut control, runner) = cyw43::new(state, pwr, spi, fw, nvram).await;
spawner.spawn(cyw43_task(runner)).unwrap();
control.init(clm).await;

// 初始化网络栈（DHCP 自动获取 IP）
let (stack, runner) = embassy_net::new(net_device, config, resources, seed);
spawner.spawn(net_task(runner)).unwrap();

// 连接 WiFi（最多重试 3 次，每次 5 秒超时）
let mut wifi_ok = false;
for _ in 0..3 {
    match control.join(WIFI_SSID, JoinOptions::new(WIFI_PASSWORD.as_bytes())).await {
        Ok(_) => { wifi_ok = true; break; }
        Err(e) => { info!("WiFi 连接失败，重试中: {:?}", e); Timer::after_secs(5).await; }
    }
}
// 等待 DHCP 获取 IP（最多 10 秒）
if wifi_ok {
    let deadline = Instant::now() + Duration::from_secs(10);
    while !stack.is_config_up() {
        if Instant::now() >= deadline { wifi_ok = false; break; }
        Timer::after_millis(100).await;
    }
}
if wifi_ok { info!("WiFi 已连接"); } else { warn!("WiFi 连接失败，基础功能不受影响"); }
```

### 8.3 天气数据来源：爬取中国天气网

本项目通过 **爬虫** 从中国天气网 `weather.com.cn` 提取实时天气数据，**免费、无需注册、无需 API Key**。

#### 工作原理

Pico 2 W 访问天气页面 `https://www.weather.com.cn/weather1d/{城市代码}.shtml`，页面 HTML 中嵌入了一段 JavaScript 变量 `observe24h_data`，包含最近 24 小时的实况观测数据：

```
页面 HTML 中嵌入的数据：

var observe24h_data = {"od":{
    "od0": "202607010900",     ← 观测时间
    "od1": "杭州",             ← 城市名
    "od2": [
        {
            "od21": "09",      ← 小时
            "od22": "24.1",    ← 温度 (°C)
            "od23": "74",      ← 风向（度数）
            "od24": "东风",    ← 风向（文字）
            "od25": "1",       ← 风力（级）
            "od26": "0.4",     ← 风速 (km/h)
            "od27": "100",     ← 湿度 (%)
            "od28": ""         ← 其他

```

代码通过流式读取 HTML 响应，搜索 `observe24h_data` 关键字，提取 JSON 并解析出温度和湿度。

#### NTP 时间同步

NTP（Network Time Protocol）用于自动校准 DS3231 的时间。

**常用 NTP 服务器：**

| 服务器 | 地址 | 说明 |
|--------|------|------|
| **国家授时中心** | `ntp.ntsc.ac.cn` | **权威，推荐使用** |
| 阿里云 | `ntp.aliyun.com` | 国内备选，延迟低 |
| 腾讯云 | `ntp.tencent.com` | 国内备选 |
| Cloudflare | `time.cloudflare.com` | 国际 |
| NTP.org | `pool.ntp.org` | 全球池 |

**NTP 工作原理：**

```
Pico 2 W                    NTP 服务器
  │                           │
  │──── UDP 请求 (端口123) ──►│
  │                           │ 服务器记录收到时间 T2
  │                           │ 服务器记录发送时间 T3
  │◄─── UDP 响应 (T1,T2,T3) ──│
  │                           │
  │ 本地记录 T1(发送) 和 T4(接收)
  │
  │  延迟 = (T4 - T1) - (T3 - T2)
  │  偏移 = ((T2 - T1) + (T3 - T4)) / 2
  │
  │  校准后时间 = 本地时间 + 偏移 + 8小时(UTC+8)
  │
  │──── 写入 DS3231 ────► DS3231 持续走时
```

代码中通过 `embassy-net` 的 UDP socket 手动实现 NTP 请求（不依赖 `sntpc` crate），获取到 UTC 时间后加 8 小时转换为北京时间，然后写入 DS3231。之后即使断网，DS3231 也会靠电池继续精确走时。

> **WiFi 状态联动：** NTP 同步和网络天气获取都有 `state.wifi_connected` 守卫，WiFi 断开时自动跳过，不影响基础功能。WiFi 状态通过 `stack.is_config_up()` 每次主循环实时检测。

### 8.6 网络天气数据结构

```rust
struct NetworkWeather {
    temp: f32,                         // 室外温度 (°C)
    humidity: f32,                     // 室外湿度 (%)
    text: heapless::String<32>,        // 天气描述（如 "晴"、"多云"、"雨"）
}
```

> **注意：** 代码中没有 `weather_code` 字段，天气描述根据温湿度自动推断（见下方映射表）。

**爬取流程（Pico 2 W）：**

```
WiFi 已连接 + stack.is_config_up() == true
  │
  ▼
DNS 解析 d1.weather.com.cn（CDN 节点，比 www 更快）
  │
  ▼
建立 TCP 连接（端口 80，10 秒超时）
  │
  ▼
发送 HTTP GET 请求
  │  GET /weather1d/{城市代码}.shtml HTTP/1.1
  │  Host: d1.weather.com.cn
  │
  ▼
流式读取 HTML 响应（最大 32KB）
  │  搜索 "observe24h_data = {" 标记
  │  提取 JSON 数据到 heapless::String<4096>
  │
  ▼
解析 JSON 中的字段
  │  "od22" → 温度 (°C)
  │  "od27" → 湿度 (%)
  │
  ▼
根据温湿度推断天气描述
  │  湿度 ≥ 90% → "雨"
  │  湿度 ≥ 80% → "阴"
  │  湿度 ≥ 60% → "多云"
  │  温度 > 35°C → "晴热"
  │  其他        → "晴"
  │
  ▼
返回 NetworkWeather → 更新到 SystemState → 显示到 TFT
```

> **流式解析：** 代码使用流式读取 + 滑动窗口匹配，只提取需要的数据（约 4KB），内存占用极小。TCP socket 关闭后自动释放 8KB 缓冲区。

### 8.7 注意事项

- Pico 2 W 只支持 **2.4GHz WiFi**，不支持 5GHz
- Pico 2 W RAM 520KB，足够使用
- 天气更新间隔建议 30 分钟以上，避免 API 限流
- WiFi 连接失败不会影响基础功能（BME280 温湿度、DS3231 时钟、表情动画、蜂鸣器、触摸都是本地的）

## 12. 常见问题排查

### 编译问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `can't find crate for std` | 没加 `#![no_std]` | 确认 main.rs 开头有 `#![no_std]` |
| `the trait bound ... is not satisfied` | embedded-hal 版本不匹配 | 检查依赖版本 |
| `overflow` | 调试模式算术溢出 | 用 `wrapping_add()` 或 release 编译 |

### 烧录问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `No connected probes were found` | Pico 未连接或驱动问题 | 检查 USB 线是否数据线；Windows 安装 Zadig 驱动 |
| `device not found` | 没有进入 BOOTSEL 模式 | **按住 BOOTSEL 再插 USB** |
| 烧录成功但没反应 | 代码 panic 了 | 用 `probe-rs attach` 查看日志 |

### 显示问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 屏幕不亮 | 接线错误或背光没接 | 检查 LED 引脚是否接 3V3，VCC 是否供电 |
| 屏幕白屏 | SPI 接线错 | 检查 SCK/MOSI/CS/DC 是否接对 |
| 颜色异常 | DC 引脚接反 | DC 高=数据，低=命令，确认接线正确 |
| 只显示部分 | 驱动芯片不匹配 | 确认屏幕是 ILI9488 而不是 ILI9341 |

### 传感器问题

| 问题 | 原因 | 解决 |
|------|------|------|
| BME280 读取失败 | I2C 地址不对 | 确认地址为 0x76（SDO 接 GND），或试 0x77 |
| 芯片 ID 不是 0x60 | 买到了 BMP280（没有湿度） | 确认芯片上写的是 BME280 不是 BMP280 |
| 气压读数为 0 | 模块版本问题 | 确认买的 BME280 不是 BME680（接口不同） |
| 值不准 | 传感器靠近热源 | 远离 Pico 芯片（Pico 发热会影响读数） |

### 时钟问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 时间不对 | 未设置初始时间 | 取消注释 `set_time` 代码烧录一次 |
| 断电后时间丢失 | 电池没电 | 更换 DS3231 模块上的 CR2032 |
| 星期几不对 | `day_of_week` 设错了 | 确认设置时的星期几正确 |

#### 🔌 第一次使用

**1. 插上电源**

用附带的 USB 线连接电脑或充电宝，屏幕会亮起，显示加载动画。

**2. 连接 WiFi**

天气站需要连接 WiFi 才能获取实时天气和自动校准时间。

WiFi 名称和密码在代码中预先配置好（`config` 模块的 `WIFI_SSID` 和 `WIFI_PASSWORD`），上电后自动连接，不需要手动操作。

> 如果 WiFi 连接失败（比如家里断网），天气站也能正常显示温度、湿度和时间，只是没有网络天气和自动对时。

**3. 时间自动校准**

连接 WiFi 后，天气站会自动从国家授时中心获取准确时间并写入 DS3231，不需要手动设置。DS3231 带纽扣电池，断电也能继续走时。

#### ⚠️ 注意事项

- **供电：** 使用附带的 USB 线连接电脑或 5V 充电器即可
- **防水：** 请勿沾水，电子元件怕水
- **清洁：** 用干布轻轻擦拭外壳和屏幕
- **电池：** 时钟模块有纽扣电池（CR2032），大约 1-2 年更换一次
- **WiFi：** 只支持 2.4GHz WiFi，不支持 5GHz
- **温度范围：** 传感器测量范围 -40°C ~ +85°C，日常使用完全够用
